Build a deterministic extension test harness

This tutorial explains a local browser workflow with setup, steps, code, and troubleshooting.

Prerequisites

Install a local browser, keep fixtures sanitized, and confirm that no internet dependency is required. These prerequisites prepare the test workflow but are not the final useful section.

Setup

Create a persistent browser profile, load the unpacked extension, start a local fixture server, and map fixture hostnames to localhost. The setup verifies that content scripts initialize, generated assets are current, and the extension can receive messages from the popup and sidebar.

const context = await chromium.launchPersistentContext(profile, { args: extensionArgs });

Step-by-step implementation

  1. Serve sanitized HTML snapshots from a local HTTP server.
  2. Launch Chromium with the unpacked extension and wait for the service worker.
  3. Send PAGEPILOT_STATUS from the extension context and assert page type, quiet mode, and best section.
  4. Trigger Jump and Next, then verify scroll movement and exact highlight placement.

The implementation section is the best target because it gives concrete commands, deterministic checks, and the complete local workflow needed to prove the runtime split.

Troubleshooting

If the browser cannot load an extension, install Playwright Chromium and rerun the test. If a fixture fails, inspect semantic DOM assertions before changing expected results.
